# 戦国AIエージェントTCG — 引き継ぎ書（Claude Code が新セッションで最初に読む）

令和から戦国に転生した軍師となり、**表＝ポケポケ風カードバトル（将軍）／裏＝AIエージェントへの命令（軍師）** を1画面で同時に回すカードゲーム。
いまは **テストプレイできるMVP** が動いており、これから **細部を作り込むフェーズ**。作者の作者（非エンジニア）がブラウザで遊んで出した感想を、コード側で直していく進め方。

## 起動と検証（まずこれ）

- 遊ぶ：`index.html` をダブルクリック（ビルド不要・オフライン可）。止まる時は `python -m http.server 8000` → `http://localhost:8000`。
- 構文：`node --check src/**/*.js`（各ファイル）。
- 自己対戦（AI対AI・クラッシュ/決着/バランス）：`node test/sim.js 30 oda takeda`（引数：戦数・自分の家・相手の家）。
- 操作テスト（jsdomで実DOM操作）：`node test/play-test.js`（25項目）。
- 実ブラウザ確認（レイアウト・見切れ・演出）：別ターミナルで `python -m http.server 8777` を立て、`node test/shot10.js`（ベンチ可視・ガタつき・スクロールを厳密計測）等。`test/shot*.js` は撮影スクリプト（スクショは OS の temp に出る）。ブラウザ側キャッシュで編集が反映されない時は `python test/nocache-server.py 8777`（no-store配信）かポート番号を変える。
- **変更したら必ず sim＋play-test を回す**。レイアウトを触ったら shot10（ベンチ可視＝はみ出し0／ガタつき0px／スクロール無）を確認。**CSS/レイアウトを触ったらスマホ幅（375px等）でも必ず確認**（2026-07-10にスマホ完全崩壊のバグを1件本番修正済み。下記「レスポンシブ（スマホ対応）」参照）。

## 固定の作り方（崩さない）

- **バニラJS・クラシックscript**。ESモジュール／バンドラ／npmビルドは使わない。各ファイルは IIFE で `window.X` にぶら下げる（`window.CARDS` `window.COMMANDS` `window.DECKS` `window.Engine` `window.UI` `window.FACTIONS` `window.CATEGORY_COLOR`）。`index.html` が `<script>` で順に読む（data → engine → ui）。
- **外部アセット禁止**。効果音は Web Audio でその場合成（`src/ui/ui.js` の `sfx()`）。画像はカードの絵だけ（`art/images/*.png`＝GPT Image 2生成の浮世絵、リポジトリに同梱）。フォント/CDN/外部URLは使わない。
- **デザイン規律**：明朝・和紙地・墨罫・アクセントは朱（伝説＝金）。影/角丸/グラデは基本なし。**全テキストは箱に収め、はみ出し・意図しない折返しを禁止**（`nowrap`/clamp）。トークンは `styles/tokens.css`。
- 描画は `UI.render()` が `#app` を丸ごと再構築する方式。**盤は必ず画面内に収める**（`.playarea` はスクロールさせない＝スクロールバー出現でガタつくため）。短い画面(660px)でもベンチ・本陣が見えることを保つ。

### レスポンシブ（スマホ対応・2026-07-10追加）

盤面(`#app`)は **1180×720の固定サイズで作られており、実画面に収まるよう常にCSS transform: scaleで縮小/拡大表示している**（`index.html`の`#viewport`＋`styles/game.css`の`#app{transform:scale(var(--ui-scale))}`＋`src/ui/ui.js`の`applyResponsiveScale()`）。
- **触ってはいけない**：`#viewport`（`position:fixed;inset:0`の外枠。実画面サイズに固定してはみ出しを吸収する）と、`applyResponsiveScale()`が`window.innerWidth`ではなく`document.documentElement.clientWidth/Height`を使っている点（狭い画面だと`#app`の固定サイズが影響して`window.innerWidth`が信頼できなくなる実測済みの罠があるため）。
- 盤面(`#app`)以外の画面（スタート/布陣/カード詳細/命令メニュー/デッキガイド等）は`#overlay`側の別レイヤーで、`max-width:92vw`等の通常のレスポンシブで対応済み。新しいオーバーレイ画面を作る時も同じパターン（`width`固定＋`max-width:9Xvw`）に合わせる。
- スマホ縦持ちは盤面デザインが横長のため縮小率が低め(幅基準で約0.3倍)になり文字が小さくなる。横持ちなら約0.5倍で見やすい。将来的にスマホ縦持ち専用レイアウトを作るなら、この「まず動く」状態から着手できる。

## 現在のルール（実装の正＝authoritative。図鑑/旧要件より新しい）

- **勝敗：相手の武将を3体討ち取れば勝ち**（`p.warlordsLost >= 3` で敗北）。
- **大名は各家の「本陣」に控える**（デッキ外）。**味方が2体討たれた時点で、3体目の砦として自動で前線へ出陣**。本陣にいる間は**無傷**（出てくる時は満タン）。大名を討てば3体目＝勝ち。手動出陣ボタンは無し。
- **出世**：たね（足軽/騎馬武者/槍足軽/弓兵/鉄砲足軽）→ 侍大将（明智など）まで。名持ち→名持ち（侍大将→大名）の進化は無い。デッキは「進化する種＋進化しない重い単体」で構成。
- **兵糧**（行動資源）：毎ターン在庫+2。武将へ送って蓄え、ワザで消費。**同じ武将へは1ターン1つまで**（前線で付けて下げて再付与する二重取りを封じる）。退き口（交代）は兵糧消費（種1/侍大将2/大名1・采配旗で無料）。
- **脳容量**（コンテキスト＝命令の資源・既定100）：自然回復は控えめ（+5/turn）。**命令を完遂しても戻らない**（撃ち得防止）。命令はメニューで**複数まとめて発令**でき、**自分の手番が終わるたびに**進行（数手番で完成）。
- 命令札14種：`/plan` `/code-review` `/security-review` `codex exec` `/compact` `--sandbox` `/agents` `/rewind` `/model` `/memory` `/deep-research`（総力調査・4手番の強力技）に加え、2026-07-11に3種追加：`/hooks`（伏兵の号令＝1回だけ反撃トラップ）`--dangerously-skip-permissions`（捨て身の一撃＝大ダメージ+自傷のリスクリワード）`/resume`（再起の号令＝討死した武将を手札に呼び戻す）。名はアルファベットのまま＝本物のClaude Code/Codexコマンドが元ネタ。
- **軍師秘伝の書＝命令はデッキごとに最大8種まで選んで使う**（2026-07-11、全家共通の固定リストから変更）。各`DECKS[id].commands`（配列）が実体。デッキ編成画面の「軍師秘伝の書」タブで自由に入れ替え可能（`src/ui/ui.js`の`deckEditCmdToggle`）。`Engine.issueCommand()`側でもデッキに入っていない命令は発注できないよう検証済み（UIだけでなくエンジンレベルで担保）。AIも自分のデッキの`commands`から選ぶ（`issueAICommands`）。
- **命令メニューはコマンドライン風**（2026-07-11、全部開いて選ぶパネル式から変更）：`/plan`のように打ち込む入力欄で絞り込み、名前とコストだけのコンパクトな一覧から、行をクリックすると詳細（効果・学び・完成手番）と選択ボタンが開く。実在するコマンドしか候補に出ない。入力欄はテキスト入力のたびに一覧部分だけ差し替える実装（`onInput`関数。フォーカス/カーソル位置を保つため、overlay全体は再描画しない）。
- 5家：織田（速攻/鉄砲）・武田（騎馬/全体・最強）・上杉（狙撃/粘り）・徳川（耐久/反撃）・豊臣（兵站/物量・軍師寄り）。スタート画面で自分と相手の家を各5択。
- **デッキ（武将札）は自由編成できる**（2026-07-11追加）：スタート画面「○○家のデッキ編成」ボタンから、無所属の共通札＋自家の武将のみで20枚を自由に組み替え可能（同名カードは3枚まで）。**他家の武将は選択肢に出ない＝属性（世界観）の制約**（`src/ui/ui.js`の`legalCardsFor()`）。編集結果は`window.DECKS[id].cards`／`.commands`を直接書き換えて保存（`DEFAULT_DECK_CARDS`／`DEFAULT_DECK_COMMANDS`に初期値を退避済みなので「初期デッキに戻す」で復元可）。永続化（次回起動後も残す）は未実装、セッション内のみ有効。
- **武将は各家5体ずつ選べる**（2026-07-11に3体→5体へ増員。侍大将のみ、大名は従来通り1体固定）。新規15体は実在の重臣から採用（例：織田＝柴田勝家/丹羽長秀/前田利家、豊臣＝加藤清正/福島正則/黒田官兵衛など）。画像は未生成（`art:null`）でテキストのフォールバック表示。
- モード：対CPU（プレイヤーだけ軍師・CPUは兵力+20）／1人二役（ホットシート）／観戦（AI対AI）。

## 次フェーズ（細部の作り込み）バックログ・既知の粗

- **バランス**（2026-07-10に5家総当たり160戦×20/組・mirrorマッチで再計測）：**先攻有利は誤差範囲**（ミラー戦で先攻/後攻の勝率に有意差なし）。実態は**家の強さの差**——織田・徳川が強い（勝率6割超）、武田は互角（5割強）、上杉・豊臣が弱い（勝率3割台）だった。豊臣は「軍師（脳容量）を回す上級者向け」でAI同士だと過小評価されるのは意図通りなので**数値では触っていない**。
  - **上杉は2026-07-10に第1回の数値調整済み**：上杉謙信(hp170→185・車懸り90→100)、直江兼続(hp130→135・義の一槍75→80)、甘粕景持(hp120→130・雨裂きの矢70→78)。理由：上杉だけ大名・侍大将とも他家よりHP/火力が明確に低かった（`毘沙門天`の与ダメ+30はコイン運＝攻撃のたび50%判定で、コンテキスト予算「軍配」の先攻後攻コインとは別物）。効果測定：非カウンター相手（織田・徳川・豊臣）への勝率が35%→48%程度に改善。**天敵の武田には引き続き弱い**（相性システム上、意図された関係の可能性が高いので今回は据え置き）。
  - 相性システム（`src/engine/engine.js`の`weakness`判定＝天敵の攻撃+20）は5家が輪になる設計（織田→武田→上杉→徳川→豊臣→織田）。今回の計測では**上杉は天敵の徳川に対して相性有利のはずが勝率48%と互角止まり**——徳川自体の地力（大名hp200・鶴翼の陣は自身の条件を自動達成し実質+40固定）が強すぎて相性差を打ち消している可能性があり、次に触るならここが本丸。
- **AIの賢さ**：CPUは素朴（安い命令を撃ち、最大ダメージで殴るだけ）。作者の依頼で**後回し合意済み**。
- **采配/陣形のアイコン未生成**：`金瘡医/早馬/伝令/狼煙/総攻撃/兵糧調達/離反工作/城/平野/大雨` は `art:null`（プロンプト未作成）。生成するなら `art/prompts/prompts.json` に足して画像化。2026-07-11に追加した新武将15体（柴田勝家・丹羽長秀・前田利家・内藤昌豊・高坂昌信・真田幸隆・前田慶次・柿崎景家・斎藤朝信・井伊直政・榊原康政・酒井忠次・加藤清正・福島正則・黒田官兵衛）も同様に`art:null`。
- **デッキ編成の積み残し**：現状セッション内のみ有効（リロードで初期デッキ・初期命令に戻る）。次回以降に本気で使うなら永続化（localStorage等）を検討。
- **射幸心は「演出だけ」採用済み**（配札フリップ・レア金光沢）。**報酬ループ/パック開封/コレクション（データ保存が要る大改修）は将来**、依頼があれば。
- **ドキュメントの鮮度**：`docs/命令札図鑑.md` はカード設定の在庫帳で、数値は最新でないことがある。**実装の正は `src/data/*.js` とゲーム内のデッキガイド/命令メニュー**。2026-07-10に照合したところ、**数値だけでなく効果そのものが違うカードが複数あった**（例：`codex-exec`図鑑は「即時1ターン」⇔実装は2ターン制／`memory`図鑑は「盤面に直接影響しない」⇔実装は全武将へ恒久バフ／`--sandbox`図鑑は「無効化」⇔実装は軽減のみ／`rewind`図鑑は「一手だけ取り消す」⇔実装は全回復／`deep-research`は図鑑では偵察札（諸国偵察）だが実装では全11種中最大威力の攻撃札「総力調査」）。図鑑を直すなら軽い作業ではないので、依頼があれば着手する。
